20.24 遇到 ``Too many open files (打開文件太多）" 的錯誤, 怎樣增加同時打開文件的允許數目？

通常有至少兩個資源限制了同時打開文件的數目: 操作系統可用的低層  ``文件說明符" 或 ``文件句柄" 的數目; 和標準 stdio  函數庫可用的 FILE 結構數目。兩個條件必須符合。在 MS-DOS 下, 可以通過設置 CONFIG.SYS, 可以控制系統文件 handle 的數目。 一些編譯器附有增加 stdio 的 FILE 結構數目的指令  (也許是一兩個源文件)。
