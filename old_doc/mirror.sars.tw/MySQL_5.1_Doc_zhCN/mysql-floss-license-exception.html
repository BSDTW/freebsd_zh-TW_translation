附錄K：MySQL FLOSS授權例外

MySQL AB關於使用MySQL客戶端庫的免費／自由軟件以及開發原始碼軟件應用程式的例外（「FLOSS例外」） 。

版本0.4，2005年9月8日

例外意圖

我們希望指定的免費／自由軟件以及開發原始碼軟件（FLOSS）應用程式能夠使用指定的、獲得GPL授權的MySQL客戶端庫（程式），而不考慮這樣的事實，即並非所有的FLOSS授權均與GNU通用公共授權（GPL）2.0版相容。

法律術語和條款

作為對GPL 2.0版所規定術語和條款的特別例外：

1.    您能自由分發完全由「程式」構成的衍生作品，以及獲得第1節所列的1個或多個授權計劃授權的1個或多個作品（每個稱為1個FLOSS作品），但應遵循下述條件：

a.    遵從GPL關於「程式」及其衍生作品的規定，除非衍生作品的可確認部分不是由「程式」衍生而來的，因而可將其本身合理地視為獨立和單獨的作品。

b.    衍生作品的所有可確認部分不是由「程式」衍生而來的，因而可將其本身合理地視為獨立和單獨的作品。

                                                    i.            對於這類作品，可按照下面所列的FLOSS授權之一予以分發，以及

                                                ii.            在相同的媒介上並依據相同的FLOSS授權，對於這些部分的目標碼或可執行形式，應與這些部分對應的機器可讀的完整的原始碼一起提供，作為這些部分對應的目標碼或可執行形式，以及

c.    依據GPL，對於在同一儲存卷或分發媒介上與「程式」或衍生作品一起提供的任何作品，如果它們不是「程式」、「衍生作品」或「FLOSS作品」的衍生物，其本身可被合理地視為獨立和單獨的作品。

如果不滿足上述條件，僅能根據GPL或MySQL AB公司其他有效授權選項的條款和條件，在此條件下拷貝、更改、分發或使用「程式」。

4.    定義

a.    這裡所使用而不是定義的術語的含義與GPL中給出的相同。

b.    衍生作品指得是根據版權法衍生的作品。

5.    適用性：該FLOSS例外適用於包含MySQL AB聲明的所有「程式」，在這類聲明中指明：「程式」可依據本FLOSS例外的條款分發。如果您建立或分發了某一作品，而該作品是遵從GPL授權的「程式」和任何其他作品的衍生作品，那麼本FLOSS例外不適用於該作品，因此，您必須將FLOSS例外聲明從作品中刪除，並在任何方面遵從GPL的要求，包括保留所有的GPL聲明。您可以從程式的副本中刪除FLOSS例外聲明，僅依照GPL的條款再分發程式的副本，但前提是程式未被您或任何第三方更改。
